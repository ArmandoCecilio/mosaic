## Overview

Links are used as navigational elements and you can use them as standalone elements or inline within bodies of text.

A Link is a fundamental navigation element. When clicked, it takes the user to a new page.

<br />
<br />
<Link link="#">Jump to top</Link>

</Tab>
<Tab id="Usage">

## Usage

- **Link text character count**: 30

### Variants

#### Regular

A regular Link is used to create text-based links anywhere.
The component supports an API that decorates the text with icons.

The text style is inherited,
| Variant | Usage | Example |
|----|----|----|
| Internal Link or Anchor | Used on a link to an internal location or anchor | <Link link="#">Jump to top</Link> |
| External Link | Used on a link to external location | <Link link="#">Link text</Link> |
| With Start/Contextual Icon | A link with icon in the start| <Link link="#" startIcon="message">Send email</Link> |
| With End/Action Icon | A link with icon in the end | <Link link="#" endIcon="message">Send email</Link> |
| Without End/Action Icon | A link without icon in the end | <Link link="#" endIcon="none" startIcon="message">Send email</Link> |
| With Styled Typography | Used with stylized typography | <Link link="#" startIcon="message"><H6>Link text</H6></Link> |

#### Document

A document Link is used to create text-based links within documents, typically Markdown.

The text style is inherited and an external icon is added for any external links.

#### Component

The `component` variant can wrap any element to make it navigate to a new page, when selected.

Unlike the `regular` variant, it does not add any icons or text decorations.

Typically this is a technical component to build other UI components, such as [Tile base](./tiles).

| Variants                | Usage                                  |
| ----------------------- | -------------------------------------- |
| With a Custom Component | You can add custom component with link |

#### Selectable

The `selectable` variant is used when the link supports a selected state.  
Typically this is a technical component to build other UI components, such as Tabs or other navigation components.

## Code sample

### LinkBase

The LinkBase component is used as a base component to create other link components.
Considered a technical component it enables us to wire a React router to our link components.

We typically do not use LinkBase directly, instead it is a technical component used to create other links.

```jsx live
<LinkBase link="#" size="medium">
  <div
    style={{
      alignItems: 'center',
      background: 'red',
      display: 'flex',
      justifyContent: 'center',
      width: '100px',
      height: '100px'
    }}
  >
    Link
  </div>
</LinkBase>
```

### LinkText

The LinkText component wraps Typography used for links, decorating it with icons depending on whether the link is internal or external.

Though it looks like the same as link, LinkText composes just the UI elements, used by the Link variant `regular`.

In order to behave as a Link, it needs to be wrapped by LinkBase.

Considered a technical component, it enables us to compose the UI of the typographic Links inside other components.

If you are looking to create a text Link, use the Standalone link instead.

#### Regular

```jsx live
<LinkText startIcon="message" link="#">
  <H3>Link text</H3>
</LinkText>
```

#### Document

```jsx live
<LinkText link="#" variant="document">
  Link text
</LinkText>
```

### Inline Links

Inline links inherit the parent style and you can create it using the `[Link text](link)` Markdown syntax.
